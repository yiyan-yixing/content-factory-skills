# 示例：都市成长小说《三十岁之前》全流程

> 展示如何用 8 层智能体 + 31 个技能创作一部完整的连载小说

## 项目信息

| 维度 | 内容 |
|------|------|
| **书名** | 《三十岁之前》 |
| **类型** | 都市成长 |
| **平台** | 番茄小说 |
| **目标** | 50 章 / 15 万字 / 日更 |
| **商业目标** | 首月付费率 > 5% / 完读率 > 40% |

```

### 2. 情绪洞察（SKILL-002）

```
核心情绪：不甘心
  ├─ 触发：被看不起、被低估、被辜负
  ├─ 升级：努力了但还是失败
  └─ 释放：逆袭成功、被认可

辅助情绪：
  ├─ 共鸣 → "这就是我"
  ├─ 心疼 → "她太不容易了"

```

```
品类：都市成长

```

## L1 设定师

### 1. 世界观（SKILL-101）

```
城市：虚构"江城"（对标成都/杭州）
核心规则：
  - 30岁是职场分水岭（公司潜规则）
  - 人脉比能力重要（表面规则）
  - 但真正逆袭靠的是选择（隐藏规则）

冲突源：
  - 年龄焦虑（30岁倒计时）
  - 阶层固化（小镇vs都市）
  - 情感vs事业（二选一困境）
```

### 2. 人物设计（SKILL-102）

| 角色 | 动机 | 致命缺陷 | 成长弧线 |
|------|------|----------|----------|
| 林晚 | 证明自己不是"小城市来的花瓶" | 不敢求助，死扛 | 学会信任→找到战友→逆袭 |
| 苏念 | 让妈妈过上好日子 | 过度讨好，迷失自我 | 认清→拒绝→独立 |
| 陆深 | 证明自己的商业眼光 | 控制欲太强 | 放手→信任→共赢 |
| 何薇 | 不被定义 | 冲动，不计后果 | 理性→策略→胜出 |

### 3. 角色关系（SKILL-103）

```
林晚 ←对立→ 何薇（同期竞争）
林晚 ←联盟→ 苏念（闺蜜）
林晚 ←张力→ 陆深（亦敌亦友）
苏念 ←暗线→ 何薇（表面不对付，实际理解）
```

### 4. 风格指南（SKILL-104）

```
调性：温暖但扎心，爽但真实
用词：现代口语，适度网络用语
句式：短句为主，情绪用排比
禁忌：不写玛丽苏、不写无脑爽、不写恋爱脑
范例："她不是不想哭，是哭完还得上班。"
反例："她泪如雨下，美得让所有人都心疼。" ❌
```

**L1 交付物**：WorldBook + CharacterCard[] + RelationMap + StyleGuide → 传给 L2

## L2 编剧

### 1. 故事弧线（SKILL-201）

```
第一幕（1-15章）：入局
  林晚入职→被小看→拼命证明→初见成效→发现规则不公平

第二幕（16-35章）：破局
  林晚遇到关键项目→与陆深合作又对立→苏念出事→何薇翻盘
  →林晚做出选择→阶段性胜利→发现更大的局

第三幕（36-50章）：定局
  终极项目→所有人立场明确→最后一战→不是赢而是成长
  →30岁前夜，所有人面对自己的选择
```

### 2. 长线规划（SKILL-202）

```
伏笔：Ch3 陆深提过的"那个人" → Ch28 揭示是何薇
伏笔：Ch7 苏念的"我不配" → Ch22 苏念被PUA真相
伏笔：Ch12 公司的"30岁优化" → Ch40 林晚直面这个规则
回收周期：5-10章内必须有推进
```

### 3. 爆点设计（SKILL-203）

```
Ch5：林晚当众被骂 → 读者心疼 → 完读率提升点
Ch15：苏念突然失联 → 制造悬念 → 追更动力
Ch25：林晚vs何薇正面冲突 → 情绪高潮 → 分享传播点
Ch35：陆深的真实身份揭露 → 反转 → 评论区爆炸
Ch45：林晚放弃最大的机会 → 意外 → 二创引爆
Ch50：30岁生日，不是结局是开始 → 余韵 → 读者自发安利
```

### 4. 连载节奏（SKILL-204）

```
每5章为一个节奏单元：
  Ch1-5：铺垫→冲突→小爆→收束→钩子
  Ch6-10：延续→升级→中爆→转折→钩子
  ...
节奏类型：张-张-爆-收-钩
禁止连续3章同节奏
```

**L2 交付物**：StoryArc + LongTermPlan + HighlightList + RhythmChart → 传给 L3

## L3 写手（Ch1 示例）

### 场景设计（SKILL-301）

```
Ch1 场景卡：
  场景1：入职第一天 — 目标：建立林晚处境 — 冲突：被安排最差工位
  场景2：午餐 — 目标：展示关系 — 冲突：没人跟她坐
  场景3：加班 — 目标：展示性格 — 冲突：发现数据错误但不敢说
  情绪线：不安→孤独→不甘
  章末钩子：她刚想开口，领导已经走了
```

### 对话写作（SKILL-302）

```
林晚："我……那个数据——"
领导（头也不回）："明天再说。"
（内心）：明天就来不及了。但她没追出去。
→ 推进剧情（数据问题留悬念）+ 揭示角色（敢想不敢说）
```

### 章节生成（SKILL-303）

```
Ch1《工位》3000字
开头钩子（前200字）：
  "林晚的工位在走廊尽头，旁边是保洁阿姨的拖把。"
  → 画面感 + 被边缘化处境 + 一句话建立人设
```

### 文风统一（SKILL-304）

```
检查：
  ✅ "她没哭，只是把拖把往旁边推了推。"（克制、真实）
  ❌ "她心如刀割，泪水在眼眶打转。"（过度煽情，违反 StyleGuide）
```

**L3 交付物**：ChapterDraft → 传给 L4

## L4 审稿

### 结构审查（SKILL-401）

```
P0（必须修）：无
P1（建议修）：Ch3 数据错误的伏笔太弱，读者可能遗忘
P2（可选）：Ch1 午餐场景可以更短
```

### 情绪强化（SKILL-402）

```
原稿：林晚看着空荡荡的餐桌，笑了笑。
强化：林晚看着空荡荡的餐桌，打开外卖APP，又关掉。她掏出自己带的饭盒。
→ 强化"省钱+被孤立"双重处境，情绪冲击更强
```

### 节奏优化（SKILL-403）

```
Ch1-5 节奏检查：
  张-张-爆-收-钩 ✅ 符合 RhythmChart
  Ch3 偏弱 → 建议加强数据错误的紧迫感
```

### 一致性检查（SKILL-404）

```
✅ 林晚人设一致：隐忍但有主见
✅ 世界观无矛盾
⚠️ Ch4 提到"部门会议"但Ch1说她在"项目组"→ 统一为"项目组会议"
```

**L4 交付物**：publish_ready = true → 传给 L5

## L5 运营

### 读者转化（SKILL-501）

```
转化漏斗：
  曝光 → 标题"30岁前被开除3次"（好奇心）
  点击 → 封面文案"她不是不行，是不被允许"（代入感）
  试读 → Ch1 前 2000 字免费（钩子够强）
  追更 → Ch5 爆点后付费墙（不得不看）
```

### 社群设计（SKILL-502）

```
读者群话题：
  - "你经历过被小看吗？"（共鸣）
  - "林晚应该说出数据错误吗？"（参与感）
  - "猜猜陆深的真实身份"（悬念互动）
```

### 留存设计（SKILL-503）

```
追更断裂点应对：
  Ch15 悬念结尾 → 下一章开头立即回应
  周末流量低 → 周五章末强钩子
  中段流失 → Ch25 大爆点拉回
```

### 传播设计（SKILL-504）

```
分享点：
  "她没哭，只是把拖把往旁边推了推" → 金句分享
  Ch25 正面对决 → 情绪高潮截图分享
  "30岁前你最想证明什么？" → 话题传播
```

**L5 交付物**：GrowthPlan[] → 传给 L6

## L6 商务

### 会员设计（SKILL-601）

```
免费：Ch1-5（建立代入感）
会员：Ch6+（追更驱动）
定价：首月 9.9 元 → 续费 19.9 元/月
权益：提前看 5 章 + 专属角色番外
```

### 付费墙设计（SKILL-602）

```
卡点：Ch5 结尾（林晚刚被骂，读者情绪高点）
免费比例：10%（5/50 章）
付费引导："林晚会开口吗？→ 会员继续看"
```

### 产品化（SKILL-603）

```
阶段1：小说连载（当前）
阶段2：角色番外 + 声音剧
阶段3：IP 授权（影视/短剧）
最小可售单元：Ch1-5 免费包（引流）+ Ch6-15 试读包（9.9元）
```

### IP 运营（SKILL-604）

```
核心 IP 资产：林晚（角色）> 故事
跨平台：番茄（首发）→ 微博（金句）→ 小红书（人设）→ 抖音（名场面）
跨形态：小说 → 声音剧 → 短剧
角色经济：林晚的"不甘心"是可复用情绪资产
```

**L6 交付物**：BusinessPlan[] → 传给 L7

  "actionable": true,

```

### 内容复盘（SKILL-702）

```
Top 3 做对了：
  1. Ch5 当众被骂 → 完读率最高
  2. 闺蜜线 → 评论最多
  3. 日更节奏 → 追更率稳定

Top 3 做错了：
  1. Ch12-14 节奏偏慢 → 跳出率上升
  2. 陆深戏份太少 → 读者吐槽
  3. 苏念线推进太慢 → 期待感下降
```

### 技能优化（SKILL-703）

```
规则更新：
  SKILL-201 v1.1：新增"每5章至少一个转折点"
  SKILL-403 v1.1：新增"连续3章无冲突需插入事件"
  SKILL-102 v1.1：新增"男主出场频率≥每3章1次"
```

### 经验沉淀（SKILL-704）

```
案例：《三十岁之前》Ch5 当众被骂
  原因：情绪强度最高点 = 完读率最高点
  规则：在第5章前设置一个"公开受辱"场景
  复用：下一部作品 Ch5 也设公开冲突场景
```

**L7 交付物**：KnowledgeBase → 反馈闭环到 L0
