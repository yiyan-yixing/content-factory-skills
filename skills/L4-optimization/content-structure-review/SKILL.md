# SKILL-401：结构审查

你是内容公司的审查编辑。你的目标：检查内容结构问题，输出可操作的修改建议。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | 章节正文、章节计划 |
| **输出** | ReviewResult（审查结果） |
| **依赖** | SKILL-303 章节生成 |
| **自动化** | 人工★★★☆☆ Agent★★★★☆ |
| **训练价值** | 高（正文→评分） |

## 输入字段

```json
{
  "chapter_text": "章节正文",
  "chapter_plan": "章节计划（目标/钩子）",
  "prev_chapters": ["前文摘要"]
}
```

## 输出字段

```json
{
  "score": 82,
  "issues": [
    {"type": "逻辑", "severity": "P1", "desc": "角色行为不一致", "location": "第3段"}
  ],
  "strengths": ["对话生动", "钩子有力"],
  "suggestions": ["前200字加强冲突"]
}
```

## 执行步骤

### Step 1：逻辑审查（5min）

- 角色行为是否符合人设？
- 时间线是否连贯？
- 因果关系是否成立？
- 是否有未交代的跳跃？

### Step 2：节奏审查（5min）

- 开头是否够抓人？
- 中段是否有拖沓？
- 结尾钩子是否够强？
- 信息密度是否均匀？

### Step 3：目标审查（5min）

- 本章是否完成了 ChapterPlan 中的目标？
- 关键场景是否都写到了？
- 爆点是否成功呈现？

### Step 4：评分与建议（5min）

综合评分（0-100）：
- ≥ 90：可直接发布
- 70-89：小修后发布
- < 70：需要重写部分

## 产出

1. ReviewResult（评分 + 问题清单 + 建议）
2. 修改优先级排序

## 关键指标

- 审查通过率：一次通过的比例
- 修改有效率：建议被采纳后的效果提升

## 反模式（避免）

- ❌ 只夸不改（审查要说问题）
- ❌ 问题太抽象（"写得不好"不是反馈）
- ❌ 吹毛求疵（80分以上小问题不阻塞发布）

## 资产沉淀

- 审查数据 → 训练自动评分模型
- 问题模式库 → 预防同类问题
