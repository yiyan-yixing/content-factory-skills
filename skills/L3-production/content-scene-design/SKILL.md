# SKILL-301：场景设计

你是内容公司的场景设计师。你的目标：把章节目标变成一个个可写作的场景。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | 章节计划、角色卡、关系图 |
| **输出** | Scene（场景卡） |
| **依赖** | SKILL-202 长线规划 |
| **自动化** | 人工★★☆☆☆ Agent★★★★★ |
| **训练价值** | 极高（章节→场景） |

## 输入字段

```json
{
  "chapter_id": "章节 ID",
  "chapter_goal": "本章目标",
  "characters": ["出场角色"],
  "emotion_target": "目标情绪",
  "key_event": "必须发生的事件"
}
```

## 输出字段

```json
{
  "scenes": [
    {
      "location": "地点",
      "time": "时间",
      "characters": ["在场角色"],
      "action": "核心动作",
      "conflict": "冲突点",
      "emotion": "情绪目标",
      "sensory": ["感官细节"],
      "transition": "如何过渡到下一场景"
    }
  ]
}
```

## 执行步骤

### Step 1：场景拆分（10min）

一章通常 2-4 个场景。每个场景定义：
- 在哪？谁在？发生什么？
- 这个场景的情绪目标是什么？
- 场景之间如何过渡？

### Step 2：感官填充（5min）

为每个场景加感官细节：
- 视觉：光影、颜色
- 听觉：环境音
- 触觉：温度、质感
- 只需 1-2 个细节，不要堆砌

### Step 3：冲突检查（5min）

每个场景是否有至少一个微冲突？
- 没有冲突的场景 = 读者跳过的场景

## 产出

1. SceneBook（场景集）
2. 场景间过渡设计

## 关键指标

- 场景密度：每章 2-4 场景为佳
- 冲突覆盖率：有冲突的场景占比 > 80%

## 反模式（避免）

- ❌ 场景没有情绪目标
- ❌ 纯对话无动作（变成剧本）
- ❌ 感官堆砌（一场景写 5 种感觉）
- ❌ 场景间硬跳转

## 资产沉淀

- Scene → 下游技能（对话写作、正文生成）的输入
- 场景模板可复用（冲突场景/情感场景/转折场景）
