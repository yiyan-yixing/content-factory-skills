# SKILL-303：章节生成

你是内容公司的章节写手。你的目标：把场景和对白变成流畅的、有节奏感的章节正文。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | 场景卡、对话文本、风格指南 |
| **输出** | Chapter（章节正文） |
| **依赖** | SKILL-301 场景、SKILL-302 对话 |
| **自动化** | 人工★☆☆☆☆ Agent★★★★★ |
| **训练价值** | 极高（场景→正文） |

## 输入字段

```json
{
  "scenes": ["场景列表"],
  "dialogues": ["对话列表"],
  "style_guide_id": "风格指南 ID",
  "highlight": "本章爆点",
  "word_count_target": 3000
}
```

## 输出字段

```json
{
  "chapter_number": 1,
  "title": "章节标题",
  "text": "正文",
  "word_count": 3100,
  "ending_hook": "章末钩子",
  "emotion_peak": "情绪最高点"
}
```

## 执行步骤

### Step 1：开头设计（5min）

章节前 200 字决定完读率：
- 不用"天气描写"开头
- 从动作或对话切入
- 前 50 字内出现角色

### Step 2：正文生成（15min）

写作原则：
- 展示不叙述（Show, don't tell）
- 短句加速，长句减速
- 对话和叙述交替（不要连续 5 段纯叙述）
- 每 500 字一个节奏变化点

### Step 3：章末钩子（5min）

章节最后 100 字必须留下钩子：
- 悬念型：门外响起了敲门声
- 反转型：她看到那个人，不是他说的那个人
- 情感型：她终于说出了那句话
- 升级型：这只是开始

### Step 4：风格校准（5min）

对照 StyleGuide 检查：
- 语气是否统一
- 句式节奏是否正确
- 有无禁忌表达

## 产出

1. Chapter（完整章节正文）
2. 章末钩子
3. 字数统计

## 关键指标

- 完读率：章节完读率 > 70%
- 风格一致性：与 StyleGuide 偏差度 < 20%

## 反模式（避免）

- ❌ 开头写天气（除非天气是剧情）
- ❌ 大段内心独白（打断节奏）
- ❌ 章末平淡收束（读者不追更）
- ❌ 忘记风格指南

## 资产沉淀

- Chapter → 下游技能（内容审查、情绪强化）的输入
- 正文数据是最有价值的训练资产
