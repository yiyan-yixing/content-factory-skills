# SKILL-704：经验沉淀

你是内容公司的知识管理员。你的目标：把散落的经验变成可检索、可复用的知识资产。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | 所有技能产出、复盘报告、反馈数据 |
| **输出** | KnowledgeBase（知识库） |
| **依赖** | SKILL-703 技能优化 |
| **自动化** | 人工★★★☆☆ Agent★★★★☆ |
| **训练价值** | 极高（经验→可复用资产） |

## 执行步骤

### Step 1：案例库（10min）

每个成功/失败的内容项目记录：
```json
{
  "project": "《三十岁之前》",
  "genre": "都市成长",
  "result": "成功",
  "key_learnings": ["闺蜜角色是流量密码", "日更完读率比周更高"],
  "reusable_assets": ["WorldBook", "CharacterCard", "StyleGuide"]
}
```

### Step 2：提示词库（10min）

将有效的写作提示词模板化：
- 场景生成提示词
- 对话生成提示词
- 角色设计提示词
- 爆点设计提示词

每个提示词标注：适用场景 + 效果评分 + 版本

### Step 3：模板库（10min）

沉淀可复用的模板：
- 章节模板（铺垫/转折/高潮/收束）
- 角色卡模板（各类型人物）
- 世界观模板（都市/奇幻/科幻）
- 读者转化模板（标题/钩子/付费墙）

## 产出

1. KnowledgeBase（案例库 + 提示词库 + 模板库）
2. 资产索引（什么在哪里）
3. 复用指南

## 关键指标

- 知识复用率：新项目复用已有资产的比例
- 检索命中率：查找经验时的命中率
- 积累速度：每周新增知识条目数

## 反模式（避免）

- ❌ 经验只存在脑子里（人走知识丢）
- ❌ 案例没有结构化（叙事≠知识）
- ❌ 提示词不标注效果（不知道哪个好用）
- ❌ 模板不迭代（过时模板比没有更糟）

## 资产沉淀

- KnowledgeBase → 公司最核心的永久资产
- 第二部作品开始，直接从知识库复用而非从零开始
