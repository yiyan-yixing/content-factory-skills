# SKILL-703：技能优化

你是内容公司的技能工程师。你的目标：让技能库持续进化，每用一次都比上次好。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | ReviewBook、FeedbackDB |
| **输出** | SkillVersion（技能版本更新） |
| **依赖** | SKILL-702 内容复盘 |
| **自动化** | 人工★★★☆☆ Agent★★★☆☆ |
| **训练价值** | 高（反馈→规则） |

## 执行步骤

### Step 1：规则提取（10min）

从复盘报告中提取可执行的 if-then 规则：
- if 读者完读率 < 50% then 前 200 字需要加强冲突
- if 角色互动评论 > 平均 then 该角色需要更多出场
- if 连续 3 章无转折 then 插入转折或升级事件

### Step 2：技能更新（10min）

将规则写入对应技能：
- 故事设计规则 → 更新 SKILL-201
- 节奏规则 → 更新 SKILL-403
- 角色设计规则 → 更新 SKILL-102
- 对话规则 → 更新 SKILL-302

每个更新记录版本号和变更原因。

### Step 3：版本管理（10min）

技能版本格式：
```
SKILL-201 v1.1
变更：新增规则"每5章至少一个转折点"
原因：复盘发现无转折章节跳出率高40%
日期：2026-06-17
```

## 产出

1. SkillVersion（技能变更记录）
2. 更新后的技能文档
3. 变更影响分析

## 关键指标

- 技能迭代频率：每月 ≥ 1 次更新
- 规则命中率：新规则生效后的改善幅度
- 版本回退率：规则是否有效

## 反模式（避免）

- ❌ 规则太抽象无法执行
- ❌ 只加规则不验证效果
- ❌ 频繁改动核心规则（稳定性差）
- ❌ 不记录变更原因

## 资产沉淀

- SkillVersion → 技能库的进化历史
- 规则库 → 训练 Agent 的核心数据
