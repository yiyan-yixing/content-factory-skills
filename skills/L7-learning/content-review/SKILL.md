# SKILL-702：内容复盘

你是内容公司的复盘官。你的目标：从已发布内容中提取经验，让下一部作品更好。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | FeedbackDB、发布数据、章节正文 |
| **输出** | ReviewBook（复盘报告） |
| **依赖** | SKILL-701 反馈收集 |
| **自动化** | 人工★★★★☆ Agent★★★☆☆ |
| **训练价值** | 高 |

## 执行步骤

### Step 1：数据回顾（15min）

- 完读率最高的 5 章（做对了什么？）
- 跳出率最高的 5 章（做错了什么？）
- 评论最多的 5 章（引发了什么？）
- 付费转化最高的 5 章（什么触发了付费？）

### Step 2：模式提取（15min）

从数据中找规律：
- 读者喜欢的情节模式是什么？
- 读者流失的节奏模式是什么？
- 哪些角色最受欢迎？为什么？
- 哪些爆点传播率最高？

### Step 3：经验沉淀（15min）

把复盘结论变成可执行规则：
- "含逆袭情节的章节完读率高 30%" → 写入故事设计规则
- "连续3章无转折跳出率飙升" → 写入节奏优化规则
- "闺蜜角色互动评论最多" → 写入人物设计规则

## 产出

1. ReviewBook（复盘报告 + 模式总结 + 经验规则）
2. Top 3 做对了 / Top 3 做错了
3. 可执行改进清单

## 关键指标

- 复盘频率：每 20 章一次
- 经验规则数：每次复盘产出 3+ 条规则
- 规则执行率：新内容是否应用了复盘规则

## 反模式（避免）

- ❌ 只看数据不提取模式
- ❌ 复盘结论太抽象（"写得更好"不是规则）
- ❌ 复盘后不更新技能库
- ❌ 只看不好不看好的（好的经验更要沉淀）

## 资产沉淀

- ReviewBook → 更新所有上游技能的经验规则
- 复盘数据 → 训练内容质量预测模型
