# SKILL-701：反馈收集

你是内容公司的数据采集员。你的目标：系统化收集读者反馈，形成可分析的数据资产。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | 平台评论、阅读数据、付费数据 |
| **输出** | FeedbackDB（反馈数据库） |
| **依赖** | 无（根技能，闭环起点） |
| **自动化** | 人工★★☆☆☆ Agent★★★★★ |
| **训练价值** | 极高 |

## 执行步骤

### Step 1：数据采集（10min）

| 数据源 | 采集内容 | 频率 |
|--------|----------|------|
| 平台评论 | 评论文本、评分 | 每日 |
| 阅读数据 | 完读率、跳出点 | 每日 |
| 付费数据 | 付费率、续费率 | 每周 |
| 社群 | 讨论热词、催更 | 每日 |

### Step 2：标签化（10min）

给每条反馈打标签：
- 情绪标签：正/负/中性
- 内容标签：角色/剧情/风格/节奏
- 行为标签：追更/流失/付费/传播

### Step 3：结构化存储（10min）

```json
{
  "id": "fb-001",
  "source": "评论区",
  "date": "2026-06-17",
  "text": "林晚太怂了求逆袭",
  "emotion": "负面（急迫）",
  "tags": ["角色", "林晚", "逆袭期待"],
  "actionable": true,
  "priority": "高"
}
```

## 产出

1. FeedbackDB（结构化反馈数据库）
2. 周度反馈摘要
3. 可操作反馈清单

## 关键指标

- 反馈采集覆盖率
- 可操作反馈比例 > 30%
- 反馈→改进的响应时间

## 反模式（避免）

- ❌ 只看好评不看差评
- ❌ 反馈散落在各处没有汇总
- ❌ 不给反馈打标签（以后无法检索）
- ❌ 收集了不分析不行动

## 资产沉淀

- FeedbackDB → 下游技能（内容复盘、技能优化）的输入
- 反馈数据是最有价值的训练资产之一
