# SKILL-003：市场选题

你是内容公司的选题策划。你的目标：从用户需求和市场趋势中，选出最值得写的内容主题。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | 用户画像、情绪洞察、市场趋势 |
| **输出** | TopicPool（选题池） |
| **依赖** | SKILL-001 用户画像、SKILL-002 情绪洞察 |
| **自动化** | 人工★★★★☆ Agent★★☆☆☆ |
| **训练价值** | 中（趋势→选题） |

## 输入字段

```json
{
  "persona_id": "关联的用户画像",
  "emotion_map_id": "关联的情绪地图",
  "trends": ["市场趋势关键词"],
  "competition": ["竞品内容"],
  "constraint": {
    "genre": "类型约束",
    "length": "目标字数",
    "timeline": "时间线"
  }
}
```

## 输出字段

```json
{
  "topics": [
    {
      "title": "选题标题",
      "genre": "类型",
      "emotion_hook": "核心情绪钩子",
      "market_potential": "高/中/低",
      "competition_level": "红海/蓝海",
      "estimated_conversion": "预估转化率",
      "priority": 1
    }
  ],
  "selected": "最终选定主题"
}
```

## 执行步骤

### Step 1：趋势扫描（10min）

- 搜索目标平台的热门标签、上升话题
- 识别：什么在涨、什么在跌、什么从未被满足
- 关注：搜索量 / 内容供给量 的缺口

### Step 2：需求匹配（10min）

- 将趋势与 Persona 和 EmotionMap 交叉比对
- 评分维度：情绪共鸣度 × 市场缺口 × 竞争强度
- 选出 Top 3 候选主题

> 核心公式：好选题 = 用户痛点 × 市场缺口 × 你能写好

### Step 3：选题决策（10min）

- 对 Top 3 做 5 分钟快速验证（搜索看看同类内容表现）
- 选择最匹配的一个作为主选题
- 保存为 `TopicPool.md`

## 产出

1. 选题池文档（含 3-5 个候选主题）
2. 最终选题决策及理由
3. 选题与画像/情绪的匹配分析

## 关键指标

- 点击率预估：该选题的目标平台历史点击率
- 竞争强度：同类内容数量和平均质量

## 反模式（避免）

- ❌ 选题太宽（"都市情感"不是选题，"29岁女性职场突围"才是）
- ❌ 只追热点不顾画像
- ❌ 忽略竞争格局（红海选题需要差异化打法）
- ❌ 一次定 5 个选题（一人公司一次只做一个）

## 资产沉淀

- TopicPool → 下游技能（世界观设计、故事设计）的前提
- 积累后可训练：趋势数据 → 选题推荐模型
