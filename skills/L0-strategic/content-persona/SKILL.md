# SKILL-001：用户画像

你是内容公司的用户研究专家。你的目标：精准定义"内容写给谁看"。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | 平台反馈、评论、行为数据 |
| **输出** | Persona（读者画像） |
| **依赖** | 无（根技能） |
| **自动化** | 人工★★★★★ Agent★☆☆☆☆ |
| **训练价值** | 高（行为→画像） |

## 输入字段

```json
{
  "reader_source": "平台名称",
  "comments": ["评论文本"],
  "behavior": {
    "age_group": "年龄区间",
    "interest": ["兴趣标签"],
    "consume_level": "消费能力",
    "reading_time": "活跃时段"
  }
}
```

## 输出字段

```json
{
  "reader_type": "读者类型标签",
  "motivation": "核心阅读动机",
  "emotion": "情绪特征",
  "consume_willingness": "付费意愿",
  "content_preference": ["偏好类型"]
}
```

## 执行步骤

### Step 1：数据收集（10min）

- 收集目标平台的用户评论、阅读行为、付费数据
- 关注：高频词、情感倾向、消费记录
- 最小样本：100 条评论 + 30 个用户行为记录

### Step 2：特征聚类（10min）

- 按年龄、兴趣、消费能力分组
- 找到最大公约数人群（不要试图覆盖所有人）
- **核心原则**：极窄人群 + 极强情绪

> 错误：面向所有年轻人写成长故事
> 正确：面向 22-30 岁一线城市女性写"向上流动焦虑"

### Step 3：画像输出（10min）

- 写出 1 个主 Persona（名字 + 标签 + 故事）
- 标注核心需求、消费意愿、阅读时段
- 保存为 `Persona.md`

## 产出

1. 读者画像文档（Persona）
2. 读者分群矩阵
3. 核心需求清单

## 关键指标

- 画像验证：用 10 条真实评论校验画像匹配度
- 留存预测：该人群 7 日留存率预估

## 反模式（避免）

- ❌ 画像太宽（"18-40岁女性"不是画像）
- ❌ 只看数据不看情绪
- ❌ 假设读者需要什么，不去验证
- ❌ 同时定义 5 个 Persona（一人公司先打一个）

## 资产沉淀

- Persona 文档 → 下游技能（情绪洞察、世界观设计、人物设计）的输入
- 积累后可训练：用户行为 → 画像 的自动生成模型
