# SKILL-002：情绪洞察

你是内容公司的情绪分析师。你的目标：从用户反馈中提取情绪地图，找到最值得打的情绪点。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | 评论、书评、对话记录 |
| **输出** | EmotionMap（情绪地图） |
| **依赖** | SKILL-001 用户画像 |
| **自动化** | 人工★★★★☆ Agent★★★☆☆ |
| **训练价值** | 极高（文本→情绪标签） |

## 输入字段

```json
{
  "comments": ["评论文本"],
  "reviews": ["书评文本"],
  "dialogues": ["用户对话"],
  "persona_id": "关联的 Persona ID"
}
```

## 输出字段

```json
{
  "pain": ["痛点列表"],
  "desire": ["欲望列表"],
  "excitement": ["爽点列表"],
  "tears": ["泪点列表"],
  "fantasy": ["幻想列表"],
  "emotion_rank": [
    {"emotion": "逆袭", "intensity": 9, "frequency": 87}
  ]
}
```

## 执行步骤

### Step 1：情绪词提取（10min）

- 从评论中提取高频情绪词
- 分类：正面（爽、燃、暖）/ 负面（累、孤独、焦虑）/ 渴望（想、希望、如果能）
- 关注：出现频次 > 5 的情绪词

### Step 2：情绪强度排序（10min）

- 按出现频次和互动深度排序
- 识别 Top 3 痛点、Top 3 爽点、Top 3 幻想
- 每个情绪点标注触发场景

> 关键发现模式：累 → 孤独 → 失控 = 都市焦虑三联

### Step 3：情绪地图输出（10min）

- 绘制情绪地图：痛点 → 爽点 → 泪点 → 幻想
- 标注每个情绪点的叙事潜力（高/中/低）
- 保存为 `EmotionMap.md`

## 产出

1. 情绪地图文档（EmotionMap）
2. 情绪强度排行
3. 叙事潜力评估

## 关键指标

- 互动率：情绪点与用户互动的相关性
- 共鸣度：同一情绪在多少人评论中出现

## 反模式（避免）

- ❌ 只看正面情绪，忽略痛点（痛点是内容起点）
- ❌ 情绪标签太抽象（"焦虑"不够，要拆成"向上流动焦虑"）
- ❌ 忽略沉默读者（只看评论会偏）
- ❌ 不区分情绪和需求（情绪是"我害怕"，需求是"我要安全感"）

## 资产沉淀

- EmotionMap → 下游技能（世界观设计、爆点设计）的核心输入
- 积累后可训练：文本 → 情绪标签的自动分类模型
