# SKILL-102：人物设计

你是内容公司的角色设计师。你的目标：创造让读者记住、关心、等待的角色。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | 世界观设定、读者画像 |
| **输出** | CharacterCard（角色卡） |
| **依赖** | SKILL-101 世界观设计 |
| **自动化** | 人工★★★★☆ Agent★★★☆☆ |
| **训练价值** | 高（设定→人物） |

## 输入字段

```json
{
  "world_id": "关联的世界观",
  "reader_segment": "目标读者分群",
  "role": "主角/配角/反派",
  "genre": "类型"
}
```

## 输出字段

```json
{
  "name": "角色名",
  "age": "年龄",
  "occupation": "职业",
  "appearance": "外貌特征（1-2个记忆点）",
  "goal": "外在目标",
  "desire": "内在欲望",
  "weakness": "致命缺陷",
  "arc": "成长弧线（从→到）",
  "voice": "语言风格标签",
  "secret": "隐藏秘密",
  "first_line": "代表性台词"
}
```

## 执行步骤

### Step 1：核心三角（15min）

每个角色定义三个维度：
- **目标**：角色想要什么（外在）
- **欲望**：角色真正需要什么（内在）
- **缺陷**：什么阻止他得到（致命弱点）

> 好角色的标志：目标≠欲望，缺陷就是成长的起点

### Step 2：成长弧线（15min）

定义角色的变化轨迹：
- 起点：缺陷状态
- 触发：什么迫使他改变
- 挣扎：反复与退缩
- 突破：最终选择
- 终点：成长后状态

> 弧线必须映射读者情绪（读者在角色身上看到自己）

### Step 3：记忆点设计（10min）

- 外貌：1-2 个独特标记（不是"很漂亮"，而是"总戴着一条褪色红围巾"）
- 口头禅：一句代表台词
- 习惯动作：一个小动作

### Step 4：角色卡输出（5min）

整理为 CharacterCard，保存到知识库。

## 产出

1. CharacterCard（主角 + 2-3 配角）
2. 角色关系简图
3. 弧线概览

## 关键指标

- 记忆度：读者能否在 3 句话内描述这个角色
- 共情度：读者是否在角色身上看到自己
- 讨论度：角色是否引发读者讨论和站队

## 反模式（避免）

- ❌ 完美角色（没有缺陷=没有故事）
- ❌ 目标和欲望相同（"她想升职因为她想升职"）
- ❌ 角色太多记不住（一部小说核心角色 ≤ 5 个）
- ❌ 角色不映射读者情绪

## 资产沉淀

- CharacterCard → 下游技能（角色关系、故事设计、对话写作）的核心输入
- 角色模板可复用（"讨好型人格主角"模板适用于多部作品）
