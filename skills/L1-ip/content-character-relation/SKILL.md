# SKILL-103：角色关系

你是内容公司的关系架构师。你的目标：用角色关系构建故事的冲突引擎。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | 角色卡列表 |
| **输出** | RelationshipGraph（关系图） |
| **依赖** | SKILL-102 人物设计 |
| **自动化** | 人工★★★☆☆ Agent★★★★☆ |
| **训练价值** | 高（角色→关系） |

## 输入字段

```json
{
  "characters": [
    {"id": "角色ID", "goal": "目标", "weakness": "缺陷"}
  ]
}
```

## 输出字段

```json
{
  "relations": [
    {
      "from": "角色A",
      "to": "角色B",
      "type": "冲突/支持/暗恋/对抗/依赖",
      "tension": "张力值 1-10",
      "evolution": "关系变化轨迹"
    }
  ],
  "core_conflict": "核心冲突",
  "power_dynamics": "权力动态"
}
```

## 执行步骤

### Step 1：冲突配对（10min）

每对角色定义一个核心张力：
- 目标冲突：两人想要同一东西
- 价值观冲突：面对同一选择立场不同
- 依赖冲突：需要对方但不想承认

> 好关系的标志：去掉任何一个人，另一个人的故事就不成立

### Step 2：权力动态（10min）

谁在上位？谁在下位？这种权力关系如何翻转？
- 职场：上司→下属
- 情感：依赖→独立
- 成长：被保护→反超

### Step 3：关系演变（10min）

定义关系的三个阶段：
- 初始状态：表面关系
- 升级触发：什么改变关系
- 最终状态：关系如何收束

## 产出

1. RelationshipGraph（关系图谱）
2. 核心冲突定义
3. 权力动态图

## 关键指标

- 张力值：每对关系的冲突密度
- 催更力：读者是否因为关系发展而追更

## 反模式（避免）

- ❌ 所有关系和谐（没有冲突就没有故事）
- ❌ 关系静态不变（关系必须演化）
- ❌ 只有一对关系有戏（配角关系也要有张力）
- ❌ 权力永远不变（翻转是高潮的前提）

## 资产沉淀

- RelationshipGraph → 下游技能（故事设计、对话写作）的核心输入
- 关系模板可复用（"师徒反目""宿敌变盟友"等经典模式）
