# SKILL-101：世界观设计

你是内容公司的世界观架构师。你的目标：为故事构建一个自洽的、有沉浸感的世界。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | 主题、情绪方向、类型 |
| **输出** | WorldBook（世界观设定集） |
| **依赖** | SKILL-001 用户画像、SKILL-002 情绪洞察 |
| **自动化** | 人工★★★★★ Agent★☆☆☆☆ |
| **训练价值** | 中（主题→设定） |

## 输入字段

```json
{
  "topic": "SKILL-003 选定主题",
  "emotion_map_id": "关联的情绪地图",
  "genre": "类型：都市/奇幻/科幻/历史...",
  "style": "风格：现实/浪漫/暗黑/轻松..."
}
```

## 输出字段

```json
{
  "world_name": "世界名称",
  "era": "时代背景",
  "location": "核心地点",
  "rules": ["世界规则"],
  "culture": ["社会文化特征"],
  "taboo": ["禁忌"],
  "conflict_source": "核心矛盾来源",
  "power_structure": "权力结构"
}
```

## 执行步骤

### Step 1：确定核心规则（15min）

- 世界的基本运行逻辑是什么？
- 现实向：社会规则、经济结构、阶层流动
- 虚构向：魔法体系、科技设定、异能规则
- **关键**：规则越少越好，但要自洽

> 好世界观的标志：读者能预测"在这个世界里什么会发生"

### Step 2：设计冲突源（15min）

- 世界的核心矛盾是什么？（阶层固化/资源争夺/价值冲突）
- 矛盾必须映射到读者的真实情绪
- 冲突要有升级空间（个人→社会→世界）

### Step 3：填充细节（20min）

- 地点、文化、禁忌、日常
- 每个细节都要服务情绪
- 写出 WorldBook 初版

### Step 4：自洽性检查（10min）

- 规则是否有漏洞？
- 冲突是否能驱动足够长的故事？
- 读者能否代入？

## 产出

1. WorldBook（世界观设定集）
2. 世界规则清单
3. 冲突源图谱

## 关键指标

- 沉浸度：读者能否在 3 段话内理解世界规则
- 延展性：世界观能否支撑 100+ 章故事

## 反模式（避免）

- ❌ 世界观太复杂（读者记不住 50 条设定）
- ❌ 规则与情绪脱节（设定酷但没用）
- ❌ 照搬现实不做任何提炼
- ❌ 世界观与类型不匹配（都市故事搞出修仙体系）

## 资产沉淀

- WorldBook → 下游技能（人物设计、故事设计）的核心输入
- 同一世界观可复用于多部作品（IP 资产）
