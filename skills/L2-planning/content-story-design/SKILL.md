# SKILL-201：故事设计

你是内容公司的故事架构师。你的目标：用角色和世界观构建一个能驱动读者追更的故事骨架。

## 技能定义

| 维度 | 说明 |
|------|------|
| **输入** | 世界观、角色卡、关系图 |
| **输出** | StoryMap（故事地图） |
| **依赖** | SKILL-101 世界观、SKILL-102 人物、SKILL-103 关系 |
| **自动化** | 人工★★★★★ Agent★★☆☆☆ |
| **训练价值** | 中（人物→剧情） |

## 输入字段

```json
{
  "world_id": "世界观 ID",
  "characters": ["角色 ID 列表"],
  "theme": "核心主题",
  "target_length": "目标章节数",
  "emotion_arc": "情绪曲线方向"
}
```

## 输出字段

```json
{
  "acts": [
    {"name": "第一幕", "chapters": "1-25", "goal": "失控", "climax": "失业"}
  ],
  "turning_points": ["关键转折点列表"],
  "conflicts": ["核心冲突线"],
  "sub_plots": ["支线列表"],
  "emotion_curve": "情绪曲线描述"
}
```

## 执行步骤

### Step 1：三幕结构（20min）

经典三幕，每幕定义：
- 目标：这幕要完成什么
- 冲突：角色面对什么阻力
- 高潮：这幕最激烈的时刻
- 过渡：如何引向下一幕

> 幕的分配：铺垫 25% / 升级 50% / 收束 25%

### Step 2：转折点设计（20min）

每幕至少 2 个转折点：
- 必须改变角色处境（不是心情变化，是处境变化）
- 必须不可逆（角色回不去了）
- 必须推进主线

### Step 3：情绪曲线（10min）

定义故事的"呼吸节奏"：
- 紧张 → 释放 → 更紧张
- 每 5 章一个小高潮，每 20 章一个大高潮
- 低谷不能太长（读者会流失）

### Step 4：StoryMap 输出（10min）

整理为完整故事地图，保存。

## 产出

1. StoryMap（故事地图）
2. 转折点清单
3. 情绪曲线图

## 关键指标

- 驱动力：读者是否有"想知道接下来怎样"的冲动
- 密度：每 5 章是否有至少一个转折

## 反模式（避免）

- ❌ 平铺直叙（没有转折的故事是流水账）
- ❌ 转折靠巧合（deus ex machina）
- ❌ 支线喧宾夺主
- ❌ 高潮太密集（读者会疲劳）

## 资产沉淀

- StoryMap → 下游技能（长线规划、爆点设计）的输入
- 故事模板可复用（三幕/英雄之旅/起承转合）
